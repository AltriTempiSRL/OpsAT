Documentación: Tokens de Color por Urgencia

Críticas (Rojo)

Errores de sistema, daños detectados, bloques operativos que exigen acción inmediata.

--red-bg (gradiente) | --red-dot (banda) | --red-text (título)

background: linear-gradient(135deg, var(--red-bg), var(--red-dot));

Alertas (Ámbar)

Advertencias, cambios de estado, tareas que requieren revisión antes de continuar.

--amber-bg (gradiente) | --amber-dot (banda) | --amber-text (título)

background: linear-gradient(135deg, var(--amber-bg), var(--amber-dot));

Éxito (Verde)

Confirmaciones, tareas completadas, hitos alcanzados. Señal positiva del flujo.

--green-bg (gradiente) | --green-dot (banda) | --green-text (título)

background: linear-gradient(135deg, var(--green-bg), var(--green-dot));

Info (Azul)

Tareas asignadas, cambios contextuales, información de operación normal.

--blue-bg (gradiente) | --blue-dot (banda) | --blue-text (título)

background: linear-gradient(135deg, var(--blue-bg), var(--blue-dot));

Características del Diseño

✓ Gradientes por tipo: `--*-bg` → `--*-dot` (135deg) en la imagen de fondo de Android.
✓ Banda lateral: 4px de ancho usando `--*-dot` como color.
✓ Títulos: usando `--*-text` para contraste en light y dark mode.
✓ Botones primarios: `background: var(--*-dot)` con texto blanco.
✓ iOS: `border-left-color: var(--*-dot)` (3px) + mismos tokens de color.

Dark Mode Automático

Todos los colores se adaptan automáticamente al tema oscuro via CSS variables. Haz clic en el toggle de arriba para verlo en acción. Cero hex hardcodeados.

[data-theme='dark'] { --red-bg: #2e1010; --red-text: #fc7272; --red-dot: #e84040; /* ... y así para todos */ }

Implementación en proxy.js + sw.js

Service Worker (sw.js) — Patrón recomendado

Al recibir una notificación push via Web Push API (message event en el service worker):

self.addEventListener('push', event => { const data = event.data.json(); const urgency = data.urgency; // 'critical', 'alert', 'success', 'info' const options = { title: data.title, body: data.body, icon: '/favicon-32.png', badge: '/badge-192.png', tag: data.taskId, // dedupe: sustituye notif previa del mismo task requireInteraction: urgency === 'critical', // NO se auto-cierra // Rich notification (Android/Chrome) image: data.imageUrl, // URL de la imagen de 300x300px (gradiente) actions: [ { action: 'open', title: 'Ver tarea', icon: '/icon-check.png' }, { action: 'dismiss', title: 'Descartar' } ], // Datos adjuntos data: { url: data.actionUrl, urgency: urgency } }; event.waitUntil(self.registration.showNotification(data.appTitle, options)); }); // Manejo de clicks en acciones self.addEventListener('notificationclick', event => { const { action, notification } = event; const data = notification.data; if (action === 'open' || !action) { clients.openWindow(data.url); } notification.close(); });

Backend (proxy.js) — Endpoint de envío

En WWP producción los endpoints reales son /api/wwp/push/test, /api/wwp/push/test-all y los envíos automáticos de createNotification().

app.post('/api/wwp/push/test-all', async (req, res) => { const { userId, taskId, urgency, title, body, actionUrl } = req.body; // urgency: 'critical' | 'alert' | 'success' | 'info' const colorMap = { critical: { bg: '#f9e8e8', dot: '#c42020' }, alert: { bg: '#fdf4e3', dot: '#cc8500' }, success: { bg: '#e2f0e8', dot: '#1e9855' }, info: { bg: '#e8f0fb', dot: '#2a68b0' } }; const colors = colorMap[urgency]; // Generar imagen de gradiente (300x300px) o usar URL precompilada const imageUrl = `data:image/svg+xml,...`; // O URL estática // Recuperar subscripción de usuario en BD const subscription = await getSubscription(userId); if (subscription) { await webpush.sendNotification(subscription, { title: title, body: body, icon: '/favicon-32.png', badge: '/badge-192.png', image: imageUrl, tag: taskId, requireInteraction: urgency === 'critical', actions: [ { action: 'open', title: 'Ver' }, { action: 'dismiss', title: 'Cerrar' } ], data: { url: actionUrl, urgency: urgency } }); } res.json({ ok: true }); });

Uso en historial.html

Cuando ocurra un evento operativo, crear la notificación interna y dejar que backend envíe Web Push con el mismo payload:

// En el listener de evento operativo const notifyPush = async (taskId, urgency, msg) => { await authFetch('/api/wwp/push/test-all', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ taskId: taskId, urgency: urgency, title: getTitleForUrgency(urgency), body: msg, actionUrl: `/historial.html#drawer=${taskId}` }) }); }; // Pick incompleto (crítico) if (missingItems.length > 0) { notifyPush(taskId, 'critical', `Faltan ${missingItems.length} artículos. Revisa ubicación ${location}.`); } // Evidencia faltante (alerta) if (!evidencePhoto) { notifyPush(taskId, 'alert', 'Debe subir foto de empaque antes de completar.'); } // Tarea completada (éxito) if (status === 'completed') { notifyPush(taskId, 'success', 'Pick validado. Despacho listo para salida.'); } // SDV nueva (info) if (eventType === 'sdv_new_pending') { notifyPush(taskId, 'info', 'Solicitud de despacho creada. Requiere revisión de dirección.'); }

Notas para aprobación

✓ Colores exactos: Todos los valores de gradiente, banda y texto provienen de las variables CSS de `historial.html` — mismo namespace, mismo dark mode automático.

✓ Consistencia visual: Las notificaciones Android y iOS usan exactamente los mismos colores que el panel interno. Un usuario que vea la notificación y luego abra la app vera la misma identidad visual.

✓ Responsive: Las tarjetas de notificación se adaptan a cualquier pantalla. Los botones son tocables en móvil (mín 36-40px).

✓ Dark mode: Toggle en el header. Light y dark usan los mismos tokens, sin fallbacks de color hexadecimal.

✓ Emojis como badge: Temporal — reemplazar por Lucide icons en implementación (ej. `AlertCircle`, `CheckCircle`, `AlertTriangle`, `Info`).
